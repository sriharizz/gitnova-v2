# Project Rules: Educational & Walkthrough Coding

- **Goal - Learn by Doing:** The user's primary goal is to understand this project perfectly. The agent must act as an educational guide, explaining the architecture, design choices, and logic behind every change.
- **Explain Tool Executions:** It is okay to execute tools (running commands, editing files), but the agent must always explain *why* the tool is being run, *what* it does, and *how* the output helps us achieve the next task.
- **Slow & Methodical Pace:** Execute exactly according to the task list in `GitNova_V3_Execution_Plan.md`, one step at a time, ensuring the user understands the current step before advancing.
